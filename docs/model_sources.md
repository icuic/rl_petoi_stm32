# Bittle model source survey

This note records candidate model sources for replacing the hand-written
`bittle_like_*` MuJoCo proxies with a higher-fidelity Petoi Bittle model.

## Preferred source: Petoi ROS description

- Repository: <https://github.com/PetoiCamp/ros_opencat>
- Default branch inspected: `ros1`
- Path: `petoi_ROS_model_docs/bittle_ros/bittle_description`
- License observed in fetched repository root: MIT
- Contents:
  - `urdf/bittle.xacro`
  - `urdf/bittle.gazebo`
  - `urdf/bittle.trans`
  - STL meshes under `meshes/`

Initial inspection shows that `bittle.xacro` contains visual meshes plus
`inertial`, `mass`, and `inertia` entries. This makes it a better starting point
than the current primitive visual MJCF for dynamics-oriented simulation work.
On the current fetch, the description contains 87 links, 86 joints, 9 movable
continuous joints, 87 inertial blocks, 87 STL meshes, and 9 transmissions.

Fetch it with:

```bash
bash scripts/fetch_petoi_model.sh
```

## Petoi non-code CAD/STL assets

- Repository: <https://github.com/PetoiCamp/NonCodeFiles>
- Path: `stl/Bittle & BittleX`
- Useful files include Bittle body, head, thigh, shank, neck, tail, and battery
  STL/STEP assets.

These files are useful for visual/collision refinement, but the ROS description
is a more direct starting point because it already provides robot topology.

## bittle-x-sim resources

- Simulator: <https://bittle-x-sim.aperobotics.io/>
- URDF: <https://bittle-x-sim.aperobotics.io/robot.urdf>

The exposed URDF appears to be generated by `onshape-to-robot` and includes
24 links, 23 joints, inertial data for all links, and 9 revolute joints
(8 leg joints plus 1 head joint). It references mesh files as
`package://assets/*.stl`.

Direct checks for common public asset URLs such as `/assets/chassis.stl`,
`/static/assets/chassis.stl`, `/assets.zip`, and `/robot.zip` returned 404.
The web bundle and WASM modules are public and contain Bittle-related symbols,
but extracting a complete reusable model from them is less maintainable than
using the Petoi ROS resources.

## Import plan

1. Fetch Petoi ROS `bittle_description` into `third_party/petoi/`.
2. Inspect the fetched model with `tools/inspect_petoi_bittle_description.py`.
3. Expand `bittle.xacro` into URDF in a generated build/cache directory.
4. Convert the URDF to MuJoCo/MJCF.
5. Keep the RL observation/action interface stable where possible:
   8 actuated leg joints, normalized action input, and explicit joint ordering.
6. Validate the imported model in stages: load-only, passive drop, stand task,
   then slow forward gait.

The first training target should be a stand/balance policy on the imported
model, not a full gait. This isolates model import and actuator tuning issues
before reward shaping for locomotion.

Current helper scripts:

```bash
bash scripts/fetch_petoi_model.sh
bash scripts/build_petoi_urdf.sh
bash scripts/validate_petoi_urdf.sh
bash scripts/build_petoi_mjcf.sh
bash scripts/validate_petoi_mjcf.sh
```

`build_petoi_urdf.sh` writes `build/petoi_bittle/robot.urdf`, copies the 87 STL
meshes beside it, names visual/collision elements for cleaner MuJoCo imports,
and raises zero diagonal inertia values to a tiny positive value. The generated
URDF currently loads in MuJoCo with 9 generalized coordinates, 9 joints, 87
geoms, and 87 meshes. It has no MuJoCo actuators yet; actuator modeling is the
next conversion step.

`build_petoi_mjcf.sh` writes `build/petoi_bittle/petoi_bittle_v0.xml` by asking
MuJoCo to import the generated URDF, wrapping the imported robot in a floating
`bittle_base` body, and adding 8 position actuators for the leg joints. The neck
joint remains passive for the first training target. The model is still a first
pass: base-body inertia and servo gains are engineering estimates that should be
calibrated before long training runs.
